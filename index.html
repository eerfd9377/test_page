3개 플랫폼 한눈에

버튼을 눌러 장단점을 펼쳐보세요.

GitHub Pages

무료 · 가장 단순

레포지토리만 있으면 끝. 개인/프로젝트 페이지에 좋음.

공식
  • 좋은점: 완전 무료, 설정 단순, Git 워크플로우와 궁합 좋음
  • 단점: 폼/서버 기능은 기본 제공이 약함(대개 외부 서비스 필요)
  • 추천: 포트폴리오/문서/간단 랜딩

Netlify

정적 + 편의기능

드래그&드롭 배포가 쉬움. 폼/리다이렉트 등 편의 기능 풍부.

공식
  • 좋은점: 배포 UX 최고(드래그&드롭/깃 연동), Netlify Forms 등 편의기능
  • 단점: 무료 플랜 한도/정책에 따라 기능 제약, 팀 규모 커지면 유료 고려
  • 추천: 빠른 데모 공개, 연락 폼 있는 랜딩

Vercel

프론트엔드 최적

Next.js 등 프레임워크에 강함. 성능/빌드/미리보기 플로우 좋음.

공식
  • 좋은점: 프리뷰 배포/성능 최적화, 프론트 프레임워크와 궁합
  • 단점: "완전 정적"만 필요하면 기능이 과할 수 있음, 정책/한도 확인 필요
  • 추천: Next.js/React 프로젝트, 프리뷰가 중요한 팀 작업
요약: 가장 단순=GitHub Pages · 가장 편함=Netlify · 프레임워크/프리뷰=Vercel

배포 전 체크리스트

정적 사이트 확인

  • index.html이 최상단에 있는지
  • CSS/JS 경로가 ./styles.css, ./script.js처럼 상대경로인지
  • 이미지/폰트 경로 깨짐 없는지

도메인/공유

  • 자동 생성된 URL로 먼저 확인
  • 공유용이면 사이트 이름/타이틀 정리
  • 필요 시 커스텀 도메인 연결
